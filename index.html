汎用LLMは学習していない領域の質問に対して、もっともらしい誤りを返すことがあります。 これを「幻覚（ハルシネーション）」と呼びます。

なぜ閉域LLMか

育てた知識は、
外に出さない。

クラウドLLMは便利ですが、業務の機密情報を外部サーバーに送るリスクがあります。 llamune はすべてをローカルで完結させ、セキュリティと競争優位性を両立します。

Security
機密情報の完全保全
顧客データも社内ノウハウも、すべて自社環境で完結。インターネットに情報が出ません。
Advantage
競合に真似できない強み
自社知識で育てたLLMは、競合が同じクラウドサービスを使っても再現できません。
Cost
従量課金コストなし
APIトークン費用が発生しません。一度構築すれば使用量に関わらず固定コストで運用できます。
構築例

用途に合わせた
3つの構成

セキュリティ要件と利用規模に合わせて選べます。

① 個人
完全隔離 — 1台のMacで完結
ネットワークに接続しないため、機密データが物理的に外部に出ません。個人の業務や社外秘データの取り扱いに最適。
セキュリティ最高（物理的完全隔離）
② チーム
閉域NW — チームで完全隔離
llamune専用のクローズドネットワークで複数人が利用。インターネット非接続でチーム全員が安全に使えます。
セキュリティ最高（ネットワーク完全隔離）
③ リモート
VPN — リモートワーク対応
社内ネットワーク上にllamuneを導入。VPN経由でリモートからも利用でき、社内セキュリティポリシーに準拠した運用が可能。
セキュリティ高（社内NW準拠＋通信暗号化）
なぜ Apple Silicon Mac なのか ▼
llamune の実行環境に Apple Silicon Mac を選んでいる理由は、そのハードウェアアーキテクチャにあります。 Apple Silicon は CPU・GPU・メモリが同一チップ上に統合された Unified Memory Architecture を採用しており、 CPU と GPU がメモリをコピーなしで共有できます。 これにより大きなモデルを効率よく展開でき、LLMの実行環境として非常に相性が良い構成になっています。
NVIDIA GPU + VM（主流構成）
  • —GPUメモリとシステムメモリが分離、転送コストが発生
  • —クラウドGPU VMはランニングコストが高い
  • —CUDA環境のセットアップが複雑
Apple Silicon Mac（llamune）
  • +Unified Memoryで大きなモデルを効率よく展開
  • +購入すれば固定コスト、消費電力も低い
  • +Apple MLXによりセットアップがシンプル

訓練方式とベースモデル

目的に合わせた
2つの訓練方式。

llamune では、目的に応じてフルファインチューニングと LoRA の2つの訓練方式を選択できます。 それぞれ使えるベースモデルが異なります。

フルファインチューニング

モデルの全パラメータを更新します。ベースモデルが持っていない新しい知識を持たせるのに適しています。非量子化モデル（bf16）のみ使用可能です。

推奨ベースモデル
Qwen2.5-3B-Instruct-bf16
Alibaba Cloud
訓練時メモリ
約 16 GB
推奨環境
Apple Silicon 32GB+
LoRA

差分アダプターのみを更新します。ベースモデルが持つ既存の知識を調整・拡張するのに適しています。量子化モデル（4bit など）でも使用可能です。

推奨ベースモデル
